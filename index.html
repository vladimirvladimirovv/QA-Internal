03 / API Testing Guidelines

Security Patterns

Verify JWT expiration, RBAC restrictions, and Rate Limiting. Ensure no PII is exposed in error payloads.

Performance

Check response time SLAs (e.g., < 300ms) and behavior under payload stress (Negative testing).

Integrity

Cross-reference API response against Database persistence using DBeaver queries.

05 / Bug Reporting Standards

Writing Guidelines

  • ❌ "The button doesn't work" — Vague, non-actionable.
  • ✅ "[Checkout] - Order Submission Fails on iOS Safari - 500 Error" — Specific, scoped.

The Standard Template

**Repro Steps:** 1. Login > 2. Open Cart > 3. Clear Storage

**Actual:** State sync fails; page hangs.

**Expected:** Redirect to Login or Empty State.

**Artifacts:** [Screenshot.png], [Network_Log.har]

**TraceID:** staging_tx_9921

06 / Investigation Protocol

1. Frontend Layer

Inspect Browser Console (F12). Is it a UI JS crash or a Network failure?

2. Network Layer

Isolate Request in Postman. Does it fail outside of the browser? Check headers/cookies.

3. Server Logs

Search TraceID in Grafana. Check Stack Traces for unhandled exceptions.

4. Data Persistence

Check DBeaver. Is the row updated? Verify foreign key constraints and triggers.
