01. QA Best Practices

Core Principles

  • Shift-Left: QA involvement during PRD/Design phase.
  • Risk-Based: Prioritize testing on high-impact features.
  • Data Management: Using clean, versioned test data sets.

Collaboration

  • Peer QA reviews for complex feature modules.
  • Supporting Devs with local testability scripts.

02. API Testing Guidelines

HTTP Status Code Reference

SOP_V2.0
Success (2xx)

200 OK: Standard Success
201 Created: New resource persistence
204 No Content: Successful deletion

Client Errors (4xx)

400 Bad Req: Validation fail
401 Unauthorized: Auth missing
403 Forbidden: RBAC failure

Server Errors (5xx)

500 Internal: Logic crash
503 Service Unavailable: Downstream failure

API Security Patterns

  • 🔒 JWT Token Expiry Validation
  • 🔒 SQL Injection in Query Params

03. Risk Areas & Mitigation

High-Risk: Payment & Identity

Likelihood: Low | Impact: Critical

  • • Concurrency in transaction locking
  • • Third-party gateway timeouts
  • • Sensitive data exposure in logs (PII)

Medium-Risk: Data Synchronization

Likelihood: High | Impact: High

  • • Eventual consistency in microservices
  • • Dead letter queue handling
  • • Race conditions during batch updates
