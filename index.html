Core Philosophy

Testing is not a phase; it's a Risk Management strategy. I focus on reducing the cost of failure by shifting quality checks to the earliest possible stage of the SDLC.

The Technical Stack

Execution Workflow

1. Requirement & Contract Review
Analyze Swagger specs. Check for breaking changes in JSON schema, data types, and mandatory fields before development begins.
2. Functional & Persistence Testing
Verify API responses via Postman and then validate that the data is correctly written to the Database using SQL queries.
3. Log Investigation & Evidence
Monitor Grafana during execution. Capture Trace IDs and payloads to provide developers with "ready-to-fix" bug reports.

Bug Reporting Excellence

Critical Impact

Payment failures, security leaks, or core service outages. Zero workaround availability.

High Priority

Major functionality broken for a specific subset of users. Workaround exists but is complex.
