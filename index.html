How I Test a Ticket (Technical Flow)

01

Discovery & Contract Review

Requirements • Swagger • Postman

Analyze the Jira ticket for edge cases. Review the Swagger/OpenAPI documentation to ensure the backend matches the frontend expectations.

CHECK: Path parameters, JSON schema types, and mandatory Auth headers.
02

Environment & Data Prep

Test Data • SQL • Mocking
Generate valid and invalid test data. Use SQL to verify the current state of the database before execution. Prepare specific user roles (RBAC) to test authorization constraints.
03

Multi-Layer Execution

Postman • DBeaver • Grafana
API

Test happy paths & negative scenarios (400, 401, 403, 422).

Database

Verify data persistence and side effects using DBeaver.

Observability

Monitor Grafana logs for hidden errors (silent 500s).

API Testing Standards

The non-negotiables for high-quality backend testing.

IDEMPOTENCY RBAC SCHEMA

Security & Auth

  • • Token expiry handling
  • • Unauthorized (401) vs Forbidden (403)
  • • SQL injection in query params

Data Integrity

  • • Payload matches DB schema
  • • No data corruption on updates
  • • Correct handling of null vs empty

System Behavior

  • • Retry logic on timeouts
  • • Idempotency on duplicate POSTs
  • • Downstream event triggers

Defect Reporting Standards

Severity: Critical

Prod down, security leak, or payment failure. No workaround.

Severity: High

Feature broken for all users, but system is online. Workaround is tedious.

Severity: Medium

Partial feature failure. Workaround exists.

Report Checklist

  • [Service] Action under Condition
  • Clear Repro Steps (1, 2, 3)
  • Expected vs Actual Results
  • Payload / IDs / Timestamps
  • Logs / Evidence (S3 or Jira)
